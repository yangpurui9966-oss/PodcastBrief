# PodcastBrief 开发计划

> **项目概述**：PodcastBrief 是一个中文播客内容提取与分析系统。用户输入播客链接或上传音频文件，系统自动获取节目资料与文字稿，通过 AI 生成结构化摘要、时间轴、内容提取等分析结果，实现「粘贴一期播客，几分钟读完一小时内容」的目标。

---

## 1. 目标与范围

### 1.1 产品目标
- 将一小时的播客内容压缩为几分钟可读完的结构化摘要
- 支持时间轴导航，用户可以跳转到感兴趣的段落
- 提取关键话题、嘉宾观点、推荐资源等结构化内容
- 支持主流中文播客平台链接解析（小宇宙、Apple Podcasts 等）

### 1.2 核心功能（MVP）
| 功能 | 描述 | 优先级 |
|------|------|--------|
| 链接解析 | 粘贴播客链接，自动抓取节目标题、简介、封面、元数据 | P0 |
| 文字稿获取 | 自动获取平台已有文字稿，或触发 AI 语音转文字 | P0 |
| 结构化摘要 | 生成节目概述、核心观点、关键话题、行动建议 | P0 |
| 时间轴分段 | 按话题/章节生成时间轴，支持一键跳转原文 | P0 |
| 内容提取 | 自动提取嘉宾推荐的书、工具、文章、金句等 | P1 |
| 文件上传 | 支持用户直接上传 MP3/WAV/M4A 音频文件 | P1 |
| 分享导出 | 生成 Markdown / 图片 / 链接形式的分享卡片 | P2 |

## 2. 技术架构

### 2.1 技术选型
| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | Next.js 14 + TypeScript + Tailwind CSS | 服务端渲染 + 现代响应式UI |
| 后端 | Next.js API Routes + Prisma + PostgreSQL | 统一在 Next.js 生态内 |
| AI 转写 | 阿里云智能语音 | 国内稳定，中文准确率高 |
| AI 分析 | DeepSeek-V3 | 成本极低，中文能力优秀 |
| 消息队列 | BullMQ (Redis) | 音频处理、AI 分析异步任务 |
| 文件存储 | 阿里云 OSS | 音频文件、封面图、生成结果 |
| 部署 | Vercel + 阿里云 ECS（Docker） | 混合部署 |

## 3. 数据模型

参见 `prisma/schema.prisma`，核心实体：

- **Podcast**：播客节目（元数据、状态、来源）
- **Transcript**：文字稿（分段、说话人、原始文本）
- **Analysis**：AI 分析结果（摘要、时间轴、资源、金句）
- **Task**：异步任务（获取元数据、转写、分析）

## 5. 关键接口设计

### 5.1 核心 API

```typescript
// 创建播客分析任务
POST /api/podcasts
Body: { url?: string } | multipart/form-data (file)
Response: { id: string; status: "pending" | "processing" | "done" | "error" }

// 获取播客详情（含分析结果）
GET /api/podcasts/:id
Response: { podcast: Podcast; transcript: Transcript; analyses: Analysis[] }

// 获取任务状态（轮询用）
GET /api/podcasts/:id/status
Response: { status: string; progress: number; error?: string }

// 重新触发分析
POST /api/podcasts/:id/reanalyze
Body: { types: AnalysisType[] }
```

## 6. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| 平台反爬导致链接解析失败 | 高 | 优先支持开放平台 API；预留手动录入兜底方案 |
| 长音频转写成本高 | 高 | 分段处理 + 缓存；优先使用平台已有文字稿 |
| AI 幻觉（生成不存在的内容） | 高 | 结构化 Prompt + 输出校验；标注「AI 生成，仅供参考」 |
| 大文件上传超时 | 中 | 分片上传 + 预签名 URL；前端进度条 |
| 多说话人识别不准确 | 中 | 基础版按停顿分段；进阶版引入 Speaker Diarization |
