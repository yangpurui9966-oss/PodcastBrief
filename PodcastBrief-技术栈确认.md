## 1. 整体架构模式

### 方案 A：全栈 Next.js 14（App Router）— **推荐**

所有功能（前端页面 + API 路由 + 服务端逻辑）统一在单个 Next.js 项目中。

| 优势 | 劣势 |
|------|------|
| 开发效率高，前后端同构（TypeScript 共享类型） | 长时间音频处理任务可能阻塞 Node.js 主线程 |
| 部署简单（Vercel 一键部署） | Vercel 免费版函数执行时间限制 10s（需要 Edge/Pro 或自建） |
| 生态成熟（Next.js 14 + App Router 已经是主流） | 服务端渲染音频文件处理需要额外配置 |

### 方案 B：前后端分离（Next.js 前端 + 独立 Node.js API 服务）

| 优势 | 劣势 |
|------|------|
| 音频处理任务独立部署，不受 Vercel 限制 | 需要维护两套代码库和部署流程 |
| 后端服务可单独扩展（ECS/ECS） | 类型共享需要额外工具（OpenAPI / tRPC） |
| 适合大型团队分工 | 开发体验不如全栈一体化 |

### 🎯 推荐决策

**阶段一~三采用方案 A（全栈 Next.js）**，原因：
1. 个人/小团队项目，开发速度优先
2. 长时间音频处理通过 **BullMQ + Redis 异步任务队列** 解耦，主 API 仅负责提交任务和返回状态
3. 部署采用 **混合方案**：前端 + API 路由部署在 Vercel，异步处理 Worker 部署在阿里云 ECS（Docker）

## 2. AI 模型选型（核心决策）

播客分析涉及两个 AI 环节：
1. **语音转写（Speech-to-Text）**：音频 → 文字稿
2. **内容分析（LLM）**：文字稿 → 结构化摘要、时间轴、提取

---

### 2.1 语音转写方案

| 方案 | 成本 | 中文准确率 | 说话人分离 | 备注 |
|------|------|------------|------------|------|
| **OpenAI Whisper API** | 低（$0.006/分钟）| ⭐⭐⭐⭐ 高 | ❌ 不支持 | 国内访问需代理；无说话人识别 |
| **阿里云智能语音（一句话识别/实时转写）** | 中（0.0015元/秒≈0.09元/分钟）| ⭐⭐⭐⭐ 高 | ⚠️ 部分支持 | 国内稳定；说话人分离需用「实时转写」 |
| **Whisper 自部署（OpenAI 开源）** | 硬件成本 | ⭐⭐⭐⭐ 高 | ❌ 不支持 | 适合高频场景，需要 GPU（如 NVIDIA T4） |
| **FunASR（阿里巴巴开源）** | 免费（自部署） | ⭐⭐⭐⭐ 高 | ✅ 支持 | 中文场景优化，支持热词、说话人分离 |

**推荐方案**：
- **MVP 阶段**：阿里云智能语音（国内稳定，无需代理）+ DeepSeek 内容分析
- **国内合规要求/长期运营**：FunASR 自部署（避免依赖海外服务）
- **混合策略**：优先用平台已有文字稿（如小宇宙提供），缺失时才触发语音转写

---

### 2.2 内容分析 LLM

播客摘要生成对模型的长上下文、理解能力、结构化输出要求较高。需要处理 1-3 万字的文字稿。

| 模型 | 上下文长度 | 中文能力 | 成本 | 国内访问 | 推荐场景 |
|------|------------|----------|------|----------|----------|
| **DeepSeek-V3** | 64K | ⭐⭐⭐⭐ | 极低（约0.0003元/1K token）| ✅ 国内稳定 | **MVP 主力模型**，性价比之王 |
| **DeepSeek-R1** | 64K | ⭐⭐⭐⭐⭐ | 低（约0.001元/1K token）| ✅ 国内稳定 | 推理能力强，适合深度分析 |
| **通义千问-Max（Qwen-Max）** | 32K | ⭐⭐⭐⭐ | 中（约0.02元/1K token）| ✅ 国内稳定 | 备选方案 |
| **通义千问-Plus（Qwen-Plus）** | 128K | ⭐⭐⭐ | 低（约0.004元/1K）| ✅ 国内稳定 | 超长文本场景 |
| **智谱 GLM-4** | 128K | ⭐⭐⭐⭐ | 中 | ✅ 国内稳定 | 备选方案 |

**关键考量**：
- 播客文字稿通常 1-3 万字（约 15K-40K tokens），DeepSeek 64K 上下文足够覆盖大部分场景
- 超长文本可以先做「分段摘要」再「合并总结」，降低单次调用 token 数
- DeepSeek API 成本极低，适合高频调用场景

### 🎯 推荐决策（已确认）

| 阶段 | 转写模型 | 分析模型 | 理由 |
|------|----------|----------|------|
| **MVP（内测）** | 阿里云智能语音 | DeepSeek-V3 | 国内零门槛，成本极低，效果优秀 |
| **公测版** | 阿里云智能语音 + FunASR | DeepSeek-V3 / R1 | 全链路国产化，成本可控 |
| **长期运营** | FunASR 自部署 | DeepSeek-V3 | 降本增效，全链路可控 |

## 3. 部署方案细化

### 3.1 推荐架构：混合部署

> 混合部署已确认。前端 + API 路由部署在 Vercel，异步 Worker 部署在阿里云 ECS（Docker）。

```
┌─────────────────────────────────────────────┐
│              Vercel（Serverless）            │
│  ┌──────────────┐  ┌──────────────────┐    │
│  │  Next.js App │  │  API Routes      │    │
│  │  (Frontend)  │  │  (短请求、CRUD)   │    │
│  └──────────────┘  └──────────────────┘    │
│                    │  /api/podcasts         │
│                    │  /api/podcasts/:id       │
│                    │  /api/podcasts/:id/status│
└────────────────────┼────────────────────────┘
                     │
                     │ 提交任务（HTTP）
                     ▼
┌─────────────────────────────────────────────┐
│         阿里云 ECS（Docker Compose）         │
│  ┌──────────────────────────────────────┐   │
│  │  BullMQ Worker 容器                  │   │
│  │  - 音频下载/转码                     │   │
│  │  - 阿里云智能语音转写                │   │
│  │  - DeepSeek AI 分析                  │   │
│  │  - 结果写回数据库                    │   │
│  └──────────────────────────────────────┘   │
│  ┌──────────────────────────────────────┐   │
│  │  Redis 容器（任务队列 + 缓存）       │   │
│  └──────────────────────────────────────┘   │
└─────────────────────────────────────────────┘
                     │
                     │ 读写数据
                     ▼
┌─────────────────────────────────────────────┐
│  阿里云 RDS（PostgreSQL）                      │
│  阿里云 OSS（音频文件 + 封面图）                │
└─────────────────────────────────────────────┘
```

### 3.2 各组件部署方式

| 组件 | 服务 | 规格建议 | 成本估算 |
|------|------|----------|----------|
| 前端 + API | Vercel Pro | 无服务器函数 | $20/月 |
| 任务 Worker | 阿里云 ECS | 2核4G（共享型） | ~150元/月 |
| 数据库 | 阿里云 RDS PostgreSQL | 2核4G 基础版 | ~200元/月 |
| 缓存 | 阿里云 Redis | 1G 主从版 | ~100元/月 |
| 对象存储 | 阿里云 OSS | 标准存储 | 按量计费 |
| 域名 + CDN | 阿里云 | 单域名 | ~50元/月 |

**MVP 阶段总成本估算：约 500-600 元/月**

### 3.3 备选：纯国内部署（阿里云 ECS + Nginx）

如果对 Vercel 有顾虑（国内访问速度、备案），可以：
- 前端构建为静态文件，部署到阿里云 CDN / OSS 静态网站托管
- API 路由部署在 ECS（Docker + PM2）
- 优点：全链路可控，备案方便
- 缺点：失去 Vercel 的 CI/CD 和 Preview 优势

## 4. 数据库与存储

### 4.1 数据库：PostgreSQL（已确认）

**为什么不用 MongoDB？**
- 播客数据关系清晰（Podcast → Transcript → Analysis），关系型数据库更自然
- Prisma 对 PostgreSQL 支持最好，类型安全、迁移自动化
- 全文搜索（后期播客文字稿检索）PostgreSQL 原生支持 `tsvector`

### 4.2 文件存储：阿里云 OSS（已确认）

**上传流程**：
1. 前端向 `/api/upload/presign` 请求预签名 URL
2. 后端生成阿里云 OSS 预签名 URL（有效期 5 分钟）
3. 前端直传 OSS（不走服务器，节省带宽）
4. 上传完成后回调后端，创建 Podcast 记录并触发分析任务
